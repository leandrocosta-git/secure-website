Secure Website with AWS

A hardened and monitored static site using AWS S3, IAM, CloudFront, GitHub Actions, and OIDC. Built as part of a 7-day secure cloud deployment challenge.

Read the full report View GitHub Repo

Least Privilege IAM

Scoped IAM roles for CI, developers, and viewers ensure write, deploy, and read-only permissions with IP/user-agent restrictions.

CI/CD with GitHub OIDC

GitHub Actions deploys directly to AWS S3 using OIDC authentication—no secrets stored, no keys rotated manually.

MFA Delete & Versioning

Critical objects are protected with S3 versioning and MFA delete to prevent tampering or accidental loss.

CloudFront + WAF

Content is served securely through CloudFront with WAF configured against OWASP Top 10 threats (SQLi, XSS, SSRF).

Secure Headers

CSP, HSTS, X-XSS-Protection, and X-Frame-Options headers are enforced via CloudFront behavior policies.

Monitoring & Logging

CloudTrail, S3 logs, CloudFront logs, and SNS alerts are enabled to track access, detect abuse, and respond to threats.

Budget & Cost Guardrails

An AWS Budget configured with email alerts. All monitoring features (CloudTrail, logs, SNS) use free tier or low-cost services.
